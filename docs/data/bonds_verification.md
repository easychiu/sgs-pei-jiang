# 緣分(bonds)外部查證差異表 —— 批34

供統籌與 Gemini 協作審核用。**本檔僅記錄差異, 不直接修改 `data/bonds.json`**(該檔為 root 資料
協作檔, 修改需經統籌/Gemini 審核後由持有寫入權限的流程處理)。

查證方式: grok(`grok -p`)與 agy(Gemini+Google搜尋, `agy -p ... --print-timeout 15m < /dev/null`)
交叉查證, 每組緣分至少一次查詢, 部分有疑義的補一次獨立查詢覆核。已由 user 戰報驗證的
「五虎上將」「魏之智(五謀臣)」不在本次查證範圍(維持現狀, 高信度)。

## 高信度差異(建議優先覆核, 多次查詢或有具體改版依據支持)

### 東吳大都督(陸遜/周瑜/呂蒙/魯肅)
- **現行 bonds.json**: 「使我軍全體的速度提升16，我軍全體主動戰法傷害提升5%。」
- **查證結果(agy)**: 速度+16 為「全體」正確; 但「主動戰法傷害+5%」官方原文限定「我軍**群體(2人)**」,
  非全體3人。
- **建議**: `effectText` 補上「群體(2人)」限定語。

### 三足鼎立(曹操/劉備/孫權)
- **現行 bonds.json**: 「戰鬥開始後前2回合，使我軍全體造成傷害提升16%。」
- **查證結果(agy)**: 官方原文為「我軍**群體(2人)**」進行兵刃攻擊和謀略攻擊的傷害提高16%, 非全體3人;
  且限定「兵刃攻擊和謀略攻擊」(即涵蓋普攻與戰法傷害, 現行描述「造成傷害」語意上一致但可更精確)。
- **建議**: 補「群體(2人)」限定。

### 江東虎臣(甘寧/太史慈/周泰/凌統)
- **現行 bonds.json**: 「使我軍全體的統率提升20，並免疫水攻。」
- **查證結果(agy)**: 「免疫水攻」為**舊版效果**, 官方平衡調整後已改為「受到普通攻擊的傷害降低5%」。
- **建議**: 高優先覆核——若確認改版屬實, 需更新 `effectText`(此為機制性改變, 非單純數值微調,
  「免疫水攻」與「普攻減傷5%」是完全不同的兩種效果)。**建議在覆核前另尋第二來源交叉確認**
  (agy單次查證, 未做二次獨立覆核)。

### 太師動亂(董卓/呂布/李儒/華雄/李傕/郭汜)
- **現行 bonds.json**: 「我軍主將的武力提升25、副將的統率提升7。」武將名單含「貂蟬」。
- **查證結果(agy)**: 
  1. 數值: 「副將統率+7」應為「副將統率提升**7%**」(百分比, 非固定點數)。
  2. 名單: 官方名單為「董卓/呂布/李儒/華雄/李傕/郭汜/**賈詡**」, **不含貂蟬**(貂蟬屬於「美人計/
     美人連環計」緣分)——現行 bonds.json 名單有「貂蟬」但缺「賈詡」。
- **建議**: 高優先覆核, 名單差異(貂蟬↔賈詡)若屬實需要連動修正 generals 資料的緣分歸屬對照,
  影響範圍較大, 建議二次獨立查證後才修改。

### 西涼霸王(董卓/馬騰/李傕/郭汜/華雄)
- **現行 bonds.json**: 「戰鬥前2回合，使我軍全體獲得必中狀態。」
- **查證結果(agy)**: 
  1. 名稱: 官方正式名稱可能是「**西涼霸雄**」而非「西涼霸王」(待覆核, 可能是agy筆誤或確有此差異)。
  2. 名單: 官方名單為「**馬超/馬騰/龐德/馬雲祿/韓遂**」, 與現行「董卓/李傕/郭汜/華雄」幾乎完全
     不重疊(只有馬騰重合)——**現行名單很可能是與「太師動亂」搞混**(董卓/李傕/郭汜/華雄同時也是
     太師動亂的成員)。
  3. 數值: 除必中狀態外，還應有「提升16點武力」，現行資料遺漏。
- **建議**: **最高優先覆核**——名單重疊錯誤(疑似兩個緣分的成員互相混淆)是本次查證中最嚴重的
  疑似錯誤, 建議立即用第二獨立來源(如官方wiki截圖)交叉確認, 這類名單級錯誤會直接影響配將
  邏輯(哪些武將同時上陣可觸發哪個緣分)。

### 美人計(貂蟬/呂布/董卓)
- **現行 bonds.json**: 「使我軍全體的武力提升14，第5回合時，使敵軍單體進入混亂狀態，持續1回合。」
- **查證結果(agy)**: 官方正式名稱可能是「**美人連環計**」, 效果為「使我軍全體武力提升14；戰鬥
  第5回合，我軍全體獲得**35%規避**，持續1回合」——「第5回合敵單體混亂」被agy認為是與貂蟬
  自身主動戰法「閉月」的混亂效果混淆, 官方緣分本身應為「我軍全體35%規避」(防守向, 非對敵方
  施加控制)。
- **建議**: 高優先覆核——「對敵方施加混亂」vs「我方自己獲得規避」是完全相反的效果方向(前者是
  控制敵方, 後者是保護己方), 若查證屬實需要修正, 且建議二次獨立來源確認(agy單次查證，因與
  「閉月」的混淆可能性有一定合理性，但也可能是agy自己誤判)。

### 黃巾之亂(張角/張寶/張梁)
- **現行 bonds.json**: 「使我軍全體的武力、智力提升9。」武將名單3人。
- **查證結果(agy)**: 數值9無誤。名單官方實際為7人陣容(張角/張寶/張梁/張燕/張曼成/管亥/卞喜,
  任選3人上陣即可觸發), 現行資料只列了3人(可能是「最常見組合範例」而非「完整名單」的簡化寫法)。
  另有SP張寶/SP張梁後續加入名單。
- **建議**: 若 bonds.json 的 `generals` 欄位語意是「完整合格名單」而非「範例組合」, 應補完整
  7人名單(+SP版本); 若只是用於「至少3人在此名單內」的判斷邏輯, 現有3人是完整名單的子集,
  可能導致某些合法組合(如張燕+管亥+卞喜)被誤判為不觸發此緣分, 屬於功能性缺陷而非純數值問題。

### 霸王討逆(孫策/周瑜/太史慈)
- **現行 bonds.json**: 「戰鬥前4回合，使我軍全體發動主動戰法、突擊戰法時，有75%機率使我軍全體
  受到的傷害降低6%。」
- **查證結果(agy)**: 機制細節有出入——官方為「每次觸發降低2%, 可疊加最多3次(封頂6%)」的疊加制,
  非「觸發後直接6%」的一次到位描述; 現行終值6%正確但疊加機制未反映。
- **建議**: 中優先——若模擬引擎目前把這條做成「一次性6%」而非「每次+2%疊加到6%封頂」，穩態
  結果趨同(多回合後都會疊滿), 但前期回合(疊層未滿時)會被高估, 建議核對 `bingshu_parsed.json`/
  `tactics_parsed.json` 有無對應建模並視情況調整為 `stack` 機制。

## 查證方法論備註

- 本次查證全數採用 grok(`grok -p`)/agy(`agy -p ... --print-timeout 15m < /dev/null`)單次或雙次
  獨立查詢, **未經 user 一手遊戲截圖交叉驗證**——AI查證工具(尤其涉及「名單」這類需要精確記憶
  多個武將組合的細節)有一定機率產生幻覺或張冠李戴(如西涼霸王/太師動亂疑似名單混淆的情形),
  信度低於本專案既有戰報實測校準。
- 「高信度差異」章節列出的項目仍建議在正式修改 `data/bonds.json` 前，用至少一個獨立第二來源
  (如遊戲內截圖、官方wiki原文截圖)交叉確認, 特別是牽涉「武將名單變更」的三筆(江東虎臣機制
  改版/太師動亂名單/西涼霸王名單)。
- 「中信度差異」多為「全體→群體(2人)」的人數限定詞補充, 或疊加機制細節, 修改風險相對較低,
  可視情況優先處理。
