# 三國志戰略版 配將引擎 - 賽季特色整合提案 (Season Integration Proposal)

本文件旨在討論如何將各 PK 賽季的「專屬特色機制」具象化為數值，並整合進入目前的配將引擎 (`sgz.py`) 中，讓「賽季 × 武將 × 特性數值」能真實影響配將評分與模擬結果。

## 一、 當前架構痛點
目前 `sgz.py` 的 `score(team, troop)` 函數，評分邏輯純粹基於：
1. 武將基礎屬性 (武/智/統/速)
2. 兵種適性 (S/A/B/C) 加成
3. 同陣營加成 (必須 3 人同陣營才能觸發 `+40` 分)
4. 戰法種類豐富度

**問題**：引擎目前為「全賽季通用盲測」，無法反映特定賽季的強勢陣容。例如在「官渡之戰」中，2+1 陣容（兩名同陣營+一名其他陣營）的實戰強度極高，但在當前引擎中會因為吃不到陣營加成而被低估。

## 二、 核心架構修改提案

我們將在引擎中引入全域參數 `CURRENT_SCENARIO`，並在 `score()` 與 `Unit()` 初始化時植入**賽季干預邏輯 (Scenario Modifiers)**。

### 1. 陣營加成機制的動態調整 (以官渡之戰為例)
*   **預設邏輯**：3 人同陣營 = 100% 陣營加成 (引擎中以 `+40` 分呈現)。
*   **官渡之戰 (pk_guandu) 邏輯**：
    *   若 3 人同陣營 = `+40` 分。
    *   **若 2 人同陣營 = 獲得 70% 陣營加成 (即 `+28` 分)**。
    *   *影響*：2+1 陣容的評分將大幅躍升，黑科技隊伍更容易進入 Top 8 推薦榜。

### 2. 養成與系統天賦的數值化
部分賽季擁有全軍增益的系統（如：個性、出身、信符），我們需要將其折算為「配將評分」或「戰鬥屬性」的固定加權：

*   **王師秉節 (信符系統)**：
    *   提案：由於信符通常提供全隊增傷/減傷或屬性，我們可以在該賽季下，給予所有 3 人完整編制的隊伍額外的 `+15` 系統分，或者允許在 CLI 輸入指定的信符來動態增加 `amp` (增傷) 或 `mitig` (減傷) 數值。
*   **北定中原 (個性系統 - 王翼/兵道/軍用)**：
    *   提案：個性系統中的「巾幗」會提升女武將屬性。若引擎判定當前為此賽季，則自動對「女性武將」額外給予 15% 的全屬性評分加權。
*   **漢焰長明 (雙武將全 S 兵種)**：
    *   提案：允許在 CLI 中指定兩名武將強制提升為 S 兵種，引擎在計算適性時直接以 S (120%) 計算。
